Key Data Flows
Critical request paths through the system
📍 Station Discovery Request
1
User opens map — client sends geo-radius query with lat/lng + optional connector filter from vehicle profile
2
API Gateway checks edge cache for identical geo-cell query (60s TTL)
3
Cache miss → Station Service runs geo-radius query, filters by connector type and station type (EV/Moto)
4
Status Aggregator pulls latest community-reported status per station from Redis
5
Community reliability scores & last-verified timestamps appended per station
6
Response cached at edge, client opens Convex subscription for community status deltas
👥 Community Status Update
1
User taps "Mark as Available / Busy / Broken" on a station — status update submitted
2
reCAPTCHA & user reputation score checked — low-trust users are rate-limited
3
Signal Processor weighs new signal against recent reports; resolves conflicts by recency & trust
4
Convex mutation writes updated status to DB, triggers reactive query invalidation
5
All subscribed clients in geo-cell receive status delta push via WebSocket
6
Contributor's reputation +XP awarded; InfluxDB logs event for reliability scoring
📸 Community Check-In & Report
1
User taps "I'm charging here" — client submits check-in with optional photo
2
reCAPTCHA & bot-score checked; user reputation score gates submission rate
3
Photo uploaded to R2, queued for Hive Moderation async (returns in <2s)
4
Community Service updates last-verified timestamp and check-in count
5
Reliability score recalculated — weighted blend of check-in frequency, recency, and reporter reputation
6
Contributor's reputation +XP awarded; operator notified if report flags issue
🗺️ Route Planning with Charge Stops
1
User enters destination + manually inputs current range — Route Service begins planning
2
Route Service calls Mapbox/OSRM for base route geometry, identifies range-gap segments
3
Station Service queries compatible stations within 5km of each gap point, filtered by user's connector selection
4
Stations ranked by community reliability score + last-verified recency + detour distance
5
Optimal stops selected, queue wait estimates added from Queue Engine
6
Compiled route handed to Mapbox Directions for turn-by-turn rendering
